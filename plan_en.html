Project implementation plan for Not Chess

I. Project Preparation

  1. Make the project structure: index.html, style.css, main.js, logic.js, ui.js, data.js, docs.html.
  2. Connect JS and CSS files in index.html.
  3. Set global constants (for example, BOARD_SIZE).

II. User Interface and Visualisation (UI)

  1. Draw a playing board 8×8 (DOM-structure).
  2. Make an inventory for the pieces: symbol + piece counter.
  3. Make buttons: Confirm, Auto-arrange, Clean, Reset.
  4. Make status message-box and visual effects.

III. Initial Setup

  1. Make the inventory clickable: choice of pieces for arrangement.
  2. Make the board clickable: placement of a selected piece and making count of the ammount of pieces.
  3. Make a piece removeable by a click.
  4. Implement players moving in turns.
  5. Make checks by pressing the button "Check": 24 pieces, correct rows.
  6. Add a 'Clean' button: remove all of player's pieces from the board.
  7. Add a 'Reset' button: full arangement reset and turn reset.
  8. Add an 'Auto-arrange' button: random arrangement.

IV. Gameplay

  1. Switch displayed pieces: hide opponents pieces behind (❓).
  2. Work on choosing a piece: check if a piece is owned by the player.
  3. Light up available turns: vacant squares and opponents pieces.
  4. Allow pieces to move to vacant squares.
  5. Work on attacks: results according to the battle system, removal of a defeated piece.
  6. Remove the (❓) from a revealed piece.
  7. Update status, pass the turn to the second player.

V. End of the Game

  1. Check if one of the win conditions has been met after every turn.
  2. Show the win announcement.
  3. Block any moving attempts.
  4. Suggest starting a new game.

VI. Special Pieces

  1. Make visual effects:
    • Mimic: blocks moving attempts of the attaker piece for 2 turns.
    • Trap: removes the attacker from the board.
    • Dominant: attaker piece becomes opponent's piece.
  2. Add activation visual effects.
  3. Show visual effect notification.

VII. Check and End

  1. Check terms of reference (docs.html): the ammount of pieces, how the King behaves, visual effects.
  2. Rewrite the game version number.
